Control-plane side path (not in raw SPI hot path):

CTRL RX FIFO -> Control Dispatch Endpoint [ADAPT firmware flow]
                |
                +-> IO Windows [ADAPT]
                    - DSHOT_IO: [LIFT/ADAPT] dshot_out + mailbox + controller
                    - PWM_IO/LED_IO/NEO_IO: [ADAPT]
                |
                +-> Status Counters [NEW]
                |
                +-> Response Payloads -> TX Mux

Key rule:
- Raw SPI hot path is RTL-only: SPI Front-End -> Parser -> CRC -> Router -> FIFOs.
- Control dispatch endpoint only handles validated CONTROL frames.
- USB serial path is an alternate transport ingress/egress feeding the same FCSP parser/router semantics.
- Both lanes share one FCSP semantic pipeline after RX arbitration.
